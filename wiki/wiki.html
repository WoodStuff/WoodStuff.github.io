The wiki is a subpage that has some information about the website. Everything had to manually be made because Github Pages unfortunately doesn't support MediaWiki.

The woodstuff.github.io wiki truly is a special wiki, as it contains no ads.

How hard is it to add a page?

It definitely takes some more work than adding another .html file.

How can I add a page?

  1. Fork the repository (here), you don't need to do this step if you've already forked it
  2. In the wiki folder, make a new html file
  3. In js/pages.js, create a new addPage() at the bottom of the chain. The parameters:
    1. The ID of the page, make this the same as the name of the HTML file you created but without the .html
    2. The display name of the page (as it appears above the article)
    3. The categories the page belongs to, see catdesc.js for a list
    4. The aliases of the page, what the user can search and still be taken to this page
  4. Copy-paste an existing page for the newly-created HTML file
  5. Set the 2 metas at the top: id should be the page id and description should be a short description
  6. Write the page
  7. Make the pull request

Some things you should know while making a page:

  • Links that lead to another wiki page shouldn't have a href attribute but p, like <a p="redsquare">
  • Templates exist (but there's only one at the moment), check the AGP square pages for an example of how you can use them
